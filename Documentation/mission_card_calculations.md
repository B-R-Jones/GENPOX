# GENPOX: Mission Card & Telemetry Calculations

This document details the calculations, equations, and rules governing the fields displayed on active harvest mission cards, as well as the underlying telemetry calculations computed during a creature's harvest expedition.

---

## 1. Active Deployed Mission Card Fields (UI)

The UI displays the active mission status card for dispatched creatures. The fields are calculated as follows:

*   **Creature Name**: Displays the name of the dispatched creature in uppercase (e.g., `m.creatureName.uppercase()`).
*   **Estimated Remaining Time (REM)**:
    $$T_{remaining} = \max(0, T_{total} - T_{elapsed})$$
    Formatted in minutes and seconds as `REM: MMm SSs`.
*   **Stall Depth (STALL)**: Displays `STALL: XX%`, representing the depth percentage at which the creature halted in the gravity well during extraction (`m.stalledDepth.toInt()`).
*   **Progress Bar**:
    $$P_{percent} = \text{coerceIn}\left(0.0, 1.0, \frac{T_{elapsed}}{T_{total}}\right)$$
    Visual progress bar drawn using Compose `Box` elements with a filled cyan color (`#00E1FF`).
*   **Tracking Status Button**:
    *   State is `🛰️ TRK_ON` (CyberGreen) if the mission is currently selected as the active tracking target (`viewModel.trackedMissionId == m.id`).
    *   State is `🛰️ TRACK` (Gray) otherwise. Clicking the button updates the active tracking focus.
*   **Harvest Phase Status Badge**:
    *   Displays `⚡ READY` (Faction-colored) if the mission harvesting/transit phase is completed (`m.isCompleted == true`).
    *   Displays `HARVESTING` (Gray) if extraction or transit is still in progress.

## 2. Active Zone Lock / Telemetry Pane Fields

When inspecting a target anomaly directly, the active mission detail pane displays real-time telemetry details:

*   **Mission Phase Status**:
    Resolved using `getPhaseLabel(activeMission.phase)` based on the active state:
    *   `"TRAVEL"` $\rightarrow$ `TRANSIT TO BOUNDARY`
    *   `"DESCENT"` $\rightarrow$ `DESCENT INTO WELL`
    *   `"HARVESTING"` $\rightarrow$ `ACTIVE EXTRACTION`
    *   `"ASCENT"` $\rightarrow$ `ASCENT TO BOUNDARY`
    *   `"TRANST_BACK"` $\rightarrow$ `TRANSIT TO BASE`
    *   `"COMPLETED"` $\rightarrow$ `EXTRACTION COMPLETED`
*   **DNA Comparison Grid**:
    A side-by-side comparative grid showing the creature's starting genetic sequence (`originalSequence`) and its current sequence (`sequence`), highlighting any structural mutation decay that occurred in the well.

## 3. Core Telemetry & Physics Calculations

Upon dispatching a sequence in `MainViewModel.kt`, physical attributes, speeds, and bounds are resolved deterministically:

### A. Boundary Radius & Anomaly Resistance
Anomalies generate a gravitational boundaries. The effective resistance scale of this boundary is influenced by synodic lunar variables:
$$R_{base} = \text{boundaryRadius} \times 0.1$$
$$R_{anom} = R_{base} \times \text{lunarResistanceMod}$$
Where:
*   $\text{lunarResistanceMod}$ is computed daily based on lunar month age cycles, ranging between $0.7$ and $1.3$.

### B. Defense & Stalled Depth
The depth to which the creature can sink before gravitational density halts its descent is determined by its defense stat and synodic resonance:
$$\text{Effective Defense} = \text{creature.defense} + \text{resonanceMod}$$
$$\text{stalledDepth} = \text{coerceIn}\left(0.0, 100.0, \frac{\text{Effective Defense}}{R_{anom}} \times 100.0\right)$$
$$\text{descentDistance} = \text{boundaryRadius} \times \frac{\text{stalledDepth}}{100.0}$$
*A higher defense allows the creature to penetrate deeper, increasing harvest accuracy.*

### C. Combined Environment Density & Drag
The surrounding environment density is a composite interference wave generated by all active overlapping anomalies:
$$\text{combinedDensity} = \sum_{i} \text{density}_i \times \cos(0.02 \times d_i + \phi_i) \times e^{-0.002 \times d_i}$$
*   $d_i$: Distance to overlapping anomaly $i$ (in feet).
*   $\phi_i$: Deterministic phase shift angle derived from the overlapping anomaly's ID.

Effective density is further offset by synodic wave cycles:
$$\text{effectiveDensity} = \text{coerceIn}(-0.33, 0.33, \text{combinedDensity} + 0.2 \times (\text{lunarPhaseScale} - 0.5))$$
If a creature possesses the **Coherence Shield** tag, drag is negated:
$$\text{finalDensity} = \begin{cases} 0.0 & \text{if Coherence Shield is active} \\ \text{effectiveDensity} & \text{otherwise} \end{cases}$$

### D. Velocities and Phase Durations
1.  **Horizontal Transit Velocity ($V_{travel}$)**:
    Based on the creature's speed stat:
    $$V_{travel} = \text{creature.speed} \times 13.5$$
2.  **Travel Duration (`travelDuration` / `transitBackDuration`)**:
    $$d_{travel} = \max\left(1, \text{round}\left(\frac{\text{anomaly.distance} - \text{boundaryRadius}}{V_{travel}}\right)\right)$$
3.  **Vertical Descent Velocity ($V_{descent}$)**:
    Applies the amplified density drag penalties and penetration friction:
    $$V_{descent} = V_{travel} \times \max(0.1, 1.0 - 2.0 \times \text{finalDensity}) \times 0.024$$
4.  **Descent Duration (`descentDuration` / `ascentDuration`)**:
    $$d_{descent} = \max\left(1, \text{round}\left(\frac{\text{descentDistance}}{V_{descent}}\right)\right)$$
5.  **Harvest Duration (`harvestDuration`)**:
    Fixed at a static extraction period of $60$ seconds.

## 4. Genetic Mutation Decay Calculations
While inside the well (during `DESCENT`, `HARVESTING`, or `ASCENT` phases), the creature's genetic sequence is susceptible to mutation:
$$\text{mutationInterval} = \max\left(1, \text{round}\left(\frac{480.0 \times 2^{-\frac{\text{stalledDepth}}{25.0}}}{\text{lunarMutationMod} \times 16.0}\right)\right)$$
*   Every $N = \text{mutationInterval}$ seconds, a single random nucleotide in the creature's 8-character DNA sequence is mutated (`A`, `G`, `T`, `C`).
*   Upon mutation, the creature's base attributes (Vitality, Attack, Defense, Speed, Faction, Type, and Weapon) are completely re-evaluated deterministically based on the mutated sequence.
